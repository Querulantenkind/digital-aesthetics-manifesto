# Creative Exercises for Designers

## Introduction

These exercises explore design within technical constraints—terminal interfaces, ASCII art, monospace typography, and the aesthetics of limitation.

Each exercise treats constraint as creative opportunity, pushing designers to work with limited palettes, fixed grids, and text-only media.

---

## Exercise 1: The 16-Color Palette Challenge

### Concept
Design a complete visual identity using only the 16 standard terminal colors.

### Materials
- Terminal emulator or color picker with hex values
- Design tool (can be terminal-based or GUI)

### Standard 16 Colors
```
Black:   #000000    Bright Black:   #808080
Red:     #800000    Bright Red:     #FF0000
Green:   #008000    Bright Green:   #00FF00
Yellow:  #808000    Bright Yellow:  #FFFF00
Blue:    #000080    Bright Blue:    #0000FF
Magenta: #800080    Bright Magenta: #FF00FF
Cyan:    #008080    Bright Cyan:    #00FFFF
White:   #C0C0C0    Bright White:   #FFFFFF
```

### Instructions

1. **Choose project**: Logo, poster, website mockup, or brand identity
2. **Restrict palette**: Use only these 16 colors (no mixing, no gradients)
3. **Design**: Create complete visual system
4. **Document**: Note which colors for what purposes

### Example Color Assignments
```
Background: Black (#000000)
Primary:    Bright Cyan (#00FFFF)
Secondary:  Yellow (#808000)
Accent:     Bright Magenta (#FF00FF)
Text:       Bright White (#FFFFFF)
Muted:      Bright Black/Gray (#808080)
Error:      Bright Red (#FF0000)
Success:    Bright Green (#00FF00)
```

### Variations

- **Monochrome**: Use only black, white, and grays
- **Color scheme**: Pick 3-4 colors, design entire system
- **Historical**: Use CGA palette (4 colors) or EGA (16 colors)
- **Custom**: Create new 16-color palette following same structure

### Reflection Prompts

- How did limited colors force creative choices?
- What patterns emerged from constraint?
- Could this work in production?
- How does this differ from unlimited color?

## Exercise 2: ASCII Art Logo Design

### Concept
Design a logo using only ASCII characters (no Unicode, no graphics).

### Materials
- Text editor with monospace font
- ASCII character set (printable characters only)

### Character Categories
```
Lines:    | - / \ + = _
Blocks:   # @ O o * .
Letters:  A-Z a-z
Numbers:  0-9
Symbols:  ! ? $ % & ~ ^ < > ( ) [ ] { }
```

### Instructions

1. **Brainstorm**: What concept/company are you designing for?
2. **Sketch**: Rough ideas on paper first
3. **Grid thinking**: Plan on character grid (each char is 1 unit)
4. **Iterate**: Try multiple versions
5. **Test scales**: View at different zoom levels

### Example

```
    ___________
   /  _______  \
  /  /       \  \
 |  |  CODE  |  |
  \  \_______/  /
   \___________/

   TECH COMPANY
```

### Variations

- **Minimalist**: Use as few characters as possible
- **Dense**: Create detailed, complex logo
- **Animated**: Design frames for simple animation
- **Responsive**: Design versions for different widths (40 col, 80 col, 120 col)

### Reflection Prompts

- How does the grid constraint affect design?
- What's lost compared to vector graphics?
- What's gained?
- Could this be a real brand identity?

## Exercise 3: Terminal UI Mockup

### Concept
Design a complete application interface for terminal display.

### Materials
- Text editor
- Box-drawing characters (optional)
- Color palette (16 colors)

### UI Elements Available
```
Borders:  ─│┌┐└┘├┤┬┴┼
Blocks:   █▓▒░
Arrows:   ←↑→↓
Bullets:  •◦▪▫
Status:   ✓✗⚠
```

### Instructions

1. **Choose app type**: Music player, email client, file manager, chat app, etc.
2. **Layout**: Sketch screen regions (header, main, sidebar, footer)
3. **Design screens**: Main view, menu, settings, about
4. **Add detail**: Buttons, lists, forms, indicators
5. **Color code**: Assign meaning to colors

### Example (Music Player)

```
┌─ NowPlaying ──────────────────────────────────────┐
│ ♫ The Midnight - Sunset                           │
│ Album: Days of Thunder (2014)                     │
│                                                    │
│ [████████████████████░░░░] 3:24 / 4:12            │
│                                                    │
│  ◀◀  ▶  ▶▶  [vol: ████████░░ 80%]                │
├───────────────────────────────────────────────────┤
│ Queue:                                            │
│  1. Current song                                  │
│  2. Next track                                    │
│  3. Another song                                  │
├───────────────────────────────────────────────────┤
│ F1:Help F2:Queue F5:Refresh F10:Quit             │
└───────────────────────────────────────────────────┘
```

### Variations

- **Different apps**: File manager, IDE, game, system monitor
- **Themes**: Design light and dark versions
- **Responsive**: Adapt to different terminal sizes
- **Interactive**: Show state changes (loading, error, success)

### Reflection Prompts

- How do terminal constraints shape UX decisions?
- What GUI patterns translate to TUI?
- What patterns don't translate?
- Is less visual complexity better or worse for users?

## Exercise 4: Monospace Typography Poster

### Concept
Design a poster using only monospace typography.

### Materials
- Text editor or design tool with monospace font
- One font family (Courier, Monaco, Consolas, etc.)

### Constraints
- Only monospace font
- Can vary: size, weight, spacing
- Can use: ASCII art, text alignment, repetition

### Instructions

1. **Choose topic**: Event, quote, concept, artwork
2. **Hierarchy**: How to show importance with monospace?
3. **Composition**: Balance, alignment, white space
4. **Execution**: Create 8.5x11" or A4 sized poster

### Example Techniques

```
SIZE for emphasis:

  BIG TEXT
  medium text
  small text


SPACING for hierarchy:

  T I T L E

  subtitle

  body text body text body text


REPETITION for texture:

  %%%%%%%%%%%%%%%%
  %%  PATTERN  %%
  %%%%%%%%%%%%%%%%


ASCII DECORATION:

  ┌───────────────┐
  │   FRAMING     │
  └───────────────┘
```

### Variations

- **One font weight**: Only regular (no bold/italic)
- **ASCII + type**: Combine ASCII art with text
- **Pure ASCII**: No typography, only ASCII characters
- **Color**: Add one or two accent colors

### Reflection Prompts

- How do you create hierarchy without font variety?
- What role does spacing play?
- Can monospace be elegant?
- What's the aesthetic of technical fonts in art?

## Exercise 6: The Progress Bar Collection

### Concept
Design 20 different progress bar styles using text characters.

### Materials
- Text editor
- Unicode block/bar characters (optional)

### Character Options
```
ASCII:     = - # * . o O
Unicode:   █ ▓ ▒ ░ ▬ ▭ ▰ ▱
Arrows:    ➤ ➔ → ⇒ ➜
Blocks:    ■ □ ▪ ▫
```

### Instructions

1. **Brainstorm**: What styles? (gradient, arrows, dots, blocks, etc.)
2. **Design 20 bars**: Each showing ~60% completion
3. **Consistent width**: All same length for comparison
4. **Label**: Name each style

### Examples

```
Classic:      [████████████░░░░░░░░] 60%
Gradient:     [████▓▓▓▓▒▒▒▒░░░░░░░░] 60%
Arrows:       [➤➤➤➤➤➤➤➤➤➤➤➤        ] 60%
Dots:         [●●●●●●●●●●●●○○○○○○○○] 60%
Blocks:       [■■■■■■■■■■■■□□□□□□□□] 60%
ASCII:        [============        ] 60%
Percentage:   [▰▰▰▰▰▰▰▰▰▰▰▰▱▱▱▱▱▱▱▱] 60%
Double:       [████████████════════] 60%
Inverted:     [░░░░░░░░░░░░████████] 60%
Chunky:       [██  ██  ██  ░░  ░░  ] 60%
```

### Variations

- **Animated**: Design spinner/loading animations
- **Vertical**: Progress bars that fill top-to-bottom
- **Circular**: ASCII/Unicode circular progress
- **Emotional**: Progress bars with personality

### Reflection Prompts

- Which styles communicate progress most clearly?
- How does style affect perceived speed?
- What's the relationship between form and function?
- Can a progress bar have personality?

## Exercise 7: Terminal Color Scheme Design

### Concept
Design a complete terminal color scheme with mood/theme in mind.

### Materials
- Color picker
- Text editor for config file
- Terminal emulator to test

### Colors to Define (16 total)
```
Background
Foreground
Cursor

Black / Bright Black
Red / Bright Red
Green / Bright Green
Yellow / Bright Yellow
Blue / Bright Blue
Magenta / Bright Magenta
Cyan / Bright Cyan
White / Bright White
```

### Instructions

1. **Choose theme**: Dark/light, warm/cool, high/low contrast, nature/tech/retro
2. **Pick base**: Start with background and foreground
3. **Define colors**: All 16 colors
4. **Test**: View code, text, interface in your scheme
5. **Refine**: Adjust for legibility and mood

### Example Themes

**Midnight Blue** (calm, coding-focused)
```
Background: #0a1628 (deep blue-black)
Foreground: #e0e7f0 (soft white)
Cursor:     #00d4ff (bright cyan)

Black:          #0a1628  Bright Black:   #405060
Red:            #ff6b6b  Bright Red:     #ff8787
Green:          #51cf66  Bright Green:   #69db7c
Yellow:         #ffd43b  Bright Yellow:  #ffe066
Blue:           #339af0  Bright Blue:    #4dabf7
Magenta:        #cc5de8  Bright Magenta: #da77f2
Cyan:           #22b8cf  Bright Cyan:    #3bc9db
White:          #adb5bd  Bright White:   #e0e7f0
```

**Warm Paper** (light, writing-focused)
```
Background: #f9f4ef (cream)
Foreground: #2e2e2e (dark gray)
Cursor:     #d9730d (burnt orange)
[... etc ...]
```

### Variations

- **Monochrome**: Only black, white, grays
- **Duotone**: Two hues only
- **Seasonal**: Spring, summer, autumn, winter themes
- **Accessibility**: High contrast, colorblind-safe

### Reflection Prompts

- How does color affect mood and productivity?
- What makes a scheme feel cohesive?
- How to balance aesthetics and legibility?
- Could your scheme work for others?

## Exercise 8: ASCII Diagram System

### Concept
Create a visual system for technical diagrams using only ASCII characters.

### Materials
- Text editor
- ASCII characters

### Diagram Types to Design
1. Flowcharts
2. Architecture diagrams
3. Network diagrams
4. Database schemas
5. Timeline/Gantt charts

### Instructions

1. **Define symbols**: What represents what?
2. **Create legend**: Document your symbol system
3. **Draw examples**: One of each diagram type
4. **Consistency**: Use same symbols across all diagrams

### Example System

```
LEGEND:
┌──┐  = Process/Action
◇     = Decision
▢     = Data/State
→     = Flow direction
╳     = Connection point
...   = Continuation
```

**Flowchart Example:**
```
        ┌──────────┐
        │  START   │
        └────┬─────┘
             │
             ▼
        ┌────────┐
        │ Input  │
        └───┬────┘
            │
            ▼
         ◇ Valid?
        ╱   │   ╲
      NO    │    YES
       │    │     │
       ▼    ▼     ▼
    Error Process Success
```

### Variations

- **UML diagrams**: Class diagrams, sequence diagrams
- **Org charts**: Hierarchical structures
- **Mind maps**: Radial layouts
- **Data viz**: Bar charts, scatter plots in ASCII

### Reflection Prompts

- What's gained by text-based diagrams?
- What's lost compared to visual tools?
- How does constraint force clarity?
- When would you use ASCII diagrams?

## Exercise 9: Pixel Font Design

### Concept
Design a complete bitmap font at 8x8 pixels.

### Materials
- Pixel editor or text editor (using █ for pixels)
- 8x8 grid

### Instructions

1. **Design alphabet**: A-Z (capitals first)
2. **Add lowercase**: a-z
3. **Numbers**: 0-9
4. **Punctuation**: . , ! ? : ; ' "
5. **Test legibility**: Type sample sentences

### Example Characters (5x7 in 8x8 grid)

```
A:        B:        C:        D:
░███░░    ████░░    ░███░░    ████░░
█░░░█░    █░░░█░    █░░░░░    █░░░█░
█░░░█░    ████░░    █░░░░░    █░░░█░
█████░    █░░░█░    █░░░░░    █░░░█░
█░░░█░    ████░░    ░███░░    ████░░
```

### Variations

- **Stylized**: Gothic, script, decorative
- **Monospace vs proportional**: Fixed or variable width?
- **Multiple sizes**: 4x6, 8x8, 16x16
- **Unicode extension**: Add symbols, emoji

### Reflection Prompts

- What's the minimum size for legibility?
- How to maintain consistency across alphabet?
- What defines the font's personality?
- Serif or sans-serif at low resolution?

## Exercise 10: Terminal Animation Storyboard

### Concept
Storyboard a short animation that plays in a terminal (ASCII/ANSI art).

### Materials
- Text editor
- Animation concept

### Instructions

1. **Concept**: What's the story/message? (5-30 seconds)
2. **Storyboard**: Draw key frames
3. **Technical plan**: How many frames? What FPS?
4. **Design frames**: Create at least 8 key frames
5. **Document**: Notes on transitions

### Example: Loading Animation (8 frames)

```
Frame 1:  [        ]  Loading...
Frame 2:  [=       ]  Loading...
Frame 3:  [==      ]  Loading...
Frame 4:  [===     ]  Loading...
Frame 5:  [====    ]  Loading...
Frame 6:  [=====   ]  Loading...
Frame 7:  [======  ]  Loading...
Frame 8:  [======= ]  Loading...
```

### Story Example: "Sunrise"

```
Frame 1:  ░░░░░░░░  (pre-dawn dark)
Frame 2:  ▁░░░░░░░  (horizon glow)
Frame 3:  ▁▂░░░░░░  (sun rising)
Frame 4:  ▁▂▃░░░░░  (sun climbing)
Frame 5:  ▁▂▃▄░░░░  (brighter)
Frame 6:  ▁▂▃▄▅░░░  (light spreading)
Frame 7:  ▁▂▃▄▅▆░░  (daytime)
Frame 8:  ▁▂▃▄▅▆▇░  (full brightness)
```

### Variations

- **Character animation**: Walking, waving, dancing
- **Weather**: Rain, snow, wind effects
- **Text effects**: Typing, fading, sliding
- **Abstract**: Patterns morphing

### Reflection Prompts

- How does frame rate affect perception?
- What's possible within terminal constraints?
- How to suggest smooth motion with discrete frames?
- Could this be functional (not just decorative)?

## Collaborative Exercises

### Group Exercise 1: Exquisite Corpse UI

**Participants**: 3-5 designers

**Process**:
1. Person A designs header
2. Person B designs main content (only sees bottom of header)
3. Person C designs sidebar (only sees edge of main)
4. Person D designs footer (only sees bottom of main)
5. Combine all pieces

**Outcome**: Surreal interface from partial information

---

### Group Exercise 2: Design System Sprint

**Participants**: 4-6 designers

**Process**:
1. Day 1: Define scope and goals
2. Day 2: Each person takes component (buttons, forms, etc.)
3. Day 3: Design components in parallel
4. Day 4: Integrate, resolve conflicts
5. Day 5: Document and present

**Outcome**: Complete design system in one week

---

### Group Exercise 3: Theme Swap

**Participants**: 2+ designers

**Process**:
1. Everyone designs terminal color scheme
2. Swap schemes (blind—don't see each other's)
3. Each person creates UI mockup using assigned scheme
4. Present: See how different designers interpret same palette

**Outcome**: Multiple perspectives on same foundation
